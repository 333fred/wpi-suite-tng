Adding a Requirement

To add a requirement

  • Click "Requirements Manager" a tab at the top
  • Click the "Create Requirement" buttonin the Home panel or right click white space in the requirements tree view and select "New Requirement"
  • Fill in the required fields. When you first create a requirement, you may not enter a Status, an Iteration, an Estimate, and an Actual. To edit some of these fields, you must save the requirement and open it again in edit view. See below for instructions on how to save.
  • Fill in the required fields, represented with a "*"
    • Name* - The name of your requirement
    • Description* - The description of your requirement
    • Priority - The importance of your requirement
    • Type - The type of requirement
    • Status - The status of the requirement (New by default, can be changed only after creation)
    • Iteration - The time segment of the requirement (Backlog by default, can be changed only after creation)
    • Estimate - The estimated amount of work required to complete the requirement.
    • Actual - The actual amount of work done in completing the requirement. The user can only edit this field after the requirement is completed.
    • Release Number - The number after the decimal place in a program's number. The release number usually indicates a minor improvement to the program.
  • Click "Save Requirement" to continue or exit by clicking "x" on the tab
  • Code example will appear under the requirement's list
